## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

### Steps

1. **Do NOT** create a public GitHub issue for the vulnerability.
2. **Report** to the security team at: `https://discord.gg/Yw4TQvfBcy`
3. Provide a detailed description of the vulnerability
4. Include steps to reproduce
5. If possible, suggest a fix

### What to Expect

- **24-48 hours**: Initial response and acknowledgment
- **1-2 weeks**: Investigation and patch development
- **After patch**: Public disclosure with credits

---

## Security Best Practices

### For Bot Users

- **Do not** share your bot token with anyone
- **Use** environment variables for sensitive data
- **Regularly update** to the latest version
- **Review** the bot's permissions before inviting

### For Developers

- **Never commit** secrets to the repository
- **Use** `.env` files for local development
- **Implement** rate limiting for API endpoints
- **Validate** all user input
- **Use** prepared statements for database queries

---

## Vulnerabilities We Care About

- Remote code execution
- SQL injection
- Authentication bypass
- Privilege escalation
- Data leaks
- Denial of service
- Cross-site scripting (XSS)

---

## Responsible Disclosure

We follow responsible disclosure practices:

1. Vulnerability is reported privately
2. We investigate and develop a fix
3. We release the fix
4. We credit the reporter (if desired)
